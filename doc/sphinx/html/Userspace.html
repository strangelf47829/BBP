Userspace Applications

BBP can load and execute applications from a persistent storage device such as an SDMMC.

Userspace applications are stored as ELF binaries (specific details on the parsing and format of ELF binaries can be found here). BBP contains various APIs to read, parse, and build ELF applications from memory or disk, without using much heap memory.

Once userspace applications have been loaded into memory, they can be executed through the virtual processor. the virtual processor is an API that can execute x86 binaries, including multi-threading and signal handling.

Several services are available to userspace applications through x86 syscalls such as garbage collection, file handling, IO services, and more. For a complete list of syscalls, please refer to (Stub).
